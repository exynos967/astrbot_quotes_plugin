# 语录 (Quotes)

一个用于 AstrBot 的 QQ 群聊语录记录插件。群友的金句、搞笑发言、经典语录，一键收藏、随机回顾。

</div>

## 功能

### 上传语录

回复目标消息后发送以下指令：

| 指令 | 说明 |
|------|------|
| `上传` | 将回复的消息记录为语录，归属原发送者 |
| `上传 @某人` | 将当前上传内容归属给指定用户；回复场景下也会覆盖默认归属 |
| `上传 QQ号` | 将当前上传内容归属给指定 QQ 号；回复场景下也会覆盖默认归属 |

- 支持纯文本、图片、图文混合消息
- 支持回复 QQ 合并转发 / 聊天记录消息并收录为聊天记录语录（仅支持“回复后上传”，并可继续用 `@某人` / `QQ号` 覆盖归属）
- 回复消息时默认归属给**被回复消息的发送者**
- 若同时写了 `@某人` 或 `QQ号`，则以指定对象为准，覆盖默认归属
- 若被记录的用户在黑名单中，将拒绝上传
- 上传的图片会进行 SHA-256 精确判重 + dhash 感知哈希近似判重，重复图片将被拒绝
- 图文混合语录会按原有顺序保存为单条语录，发送时通过图文合并消息发送
- 聊天记录语录会保留顺序与常见媒体，发送时恢复为聊天记录格式
- 聊天记录中的图片、语音、视频、文件会优先保存到本地；无法保存的媒体会降级为占位符，不影响整条语录保存

### 随机语录

| 指令 | 说明 |
|------|------|
| `语录` | 随机发送一条语录 |
| `语录 @某人` | 仅随机指定用户的语录 |
| `语录 QQ号` | 仅随机指定 QQ 号的语录 |

- 语录包含图片时直接发送原图，无图片则渲染语录卡片
- 当语录同时包含文字和图片时，会按顺序发送图文混合消息链
- 当语录为聊天记录语录时，会按聊天记录格式发送合并转发节点消息
- 开启**全局模式**后，随机范围覆盖所有群/私聊；否则仅限当前会话
- 没有语录时不回复（静默）

### 删除语录

回复机器人刚发出的随机语录消息后发送 `删除`（或 `删除语录`）。

删除权限由 `delete_permission` 配置项控制：

| 权限级别 | 可删除人员 |
|----------|------------|
| `群员` | 所有人 |
| `管理员` | 群管理员 + 群主 + Bot 管理员 |
| `群主` | 群主 + Bot 管理员 |
| `Bot管理员` | 仅 Bot 管理员 |

### 戳一戳触发

对 Bot 发送"戳一戳"（Poke）时，按配置的概率随机发送一条语录。

- 需开启 `poke_enabled` 配置项
- 触发概率由 `poke_probability` 控制（0-100）
- 支持群白名单与黑名单

## 配置项

在 AstrBot 插件设置页面中修改：

| 配置项 | 类型 | 默认值 | 说明 |
|--------|------|--------|------|
| `storage` | 字符串 | `""` | 数据存储根目录，留空使用 AstrBot 默认目录 |
| `global_mode` | 布尔 | `false` | 全局模式：开启后随机语录在所有群/私聊间共享 |
| `delete_permission` | 字符串 | `"管理员"` | 删除权限：群员 / 管理员 / 群主 / Bot管理员 |
| `blacklist` | 列表 | `[]` | 拒绝上传语录的用户 QQ 号列表 |
| `avatar_provider` | 字符串 | `"qlogo"` | 头像来源：`qlogo`（QQ 头像接口）或 `platform`（需 Napcat 权限） |
| `image.width` | 整数 | `1280` | 语录图片宽度（px） |
| `image.height` | 整数 | `427` | 语录图片高度（px） |
| `image.bg_color` | 字符串 | `"#000"` | 语录卡片右侧背景色 |
| `image.text_color` | 字符串 | `"#fff"` | 语录卡片文字颜色 |
| `image.font_family` | 字符串 | 系统默认 | 语录卡片字体 |
| `image_signature_use_group` | 布尔 | `false` | 语录图片签名使用群名片而非 QQ 昵称 |
| `poke_enabled` | 布尔 | `false` | 启用戳一戳触发随机语录 |
| `poke_probability` | 整数 | `20` | 戳一戳触发概率（0-100） |
| `poke_group_whitelist` | 列表 | `[]` | 戳一戳触发的群白名单（留空不限制） |
| `poke_group_blacklist` | 列表 | `[]` | 戳一戳触发的群黑名单 |

## 存储结构

插件数据默认保存在 `data/plugin_data/quotes/`：

```
quotes/
└── groups/
    └── {session_key}/          # 群号 或 private_QQ号
        ├── quotes.json         # 语录数据
        ├── image_index.json    # 图片资源索引
        ├── media_index.json    # 语音/视频/文件资源索引
        ├── images/             # 图片文件
        ├── media/              # 聊天记录中的语音/视频/文件
        └── cache/              # 渲染缓存
```

- 每个群/私聊会话拥有独立的数据目录
- `global_mode` 仅影响查询范围，不改变物理落盘目录
- 聊天记录语录中的图片仍进入 `images/`，语音/视频/文件进入 `media/`，都按会话独立管理
- 首次启动时自动迁移旧版单文件数据格式，并备份为 `quotes.json.bak`

## 重复图片检测

上传图片时进行双重判重：

1. **SHA-256 精确判重** — 完全相同的图片直接拒绝
2. **dhash 感知哈希近似判重** — 识别经过压缩、缩放、轻微编辑的同图

命中重复时会提示：`语录图片已存在`

## 依赖

- `httpx >= 0.24` — 异步 HTTP 客户端
- `Pillow >= 10.0` — 图片处理（感知哈希计算）

## 说明

- 基于 NapCat/OneBot v11 平台，调用 `get_msg`、`get_image`、`get_group_member_info` 等接口
- 语录卡片渲染使用 AstrBot 内置浏览器渲染能力
- 头像来源：`https://q1.qlogo.cn/g?b=qq&nk={qq}&s=640`
